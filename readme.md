Приложение позволяет выполнить синхронизацию файла с webdav-сервером. 

config.go: 
1. Объявлена структура данных и их json-тегов.
2. К структуре Config написан метод Load(), принимающий путь к json файлу конфигурации. 
3. Обработка ошибок включает в себя случаи чтения и открытия файла json и преобразование его в структуру в коде.
4. Метод Load() загружает данные из json файла и формирует структуру для дальнейшей работы.

net.go:
1. Реализована функция CheckInternetConnection(), которая проверяет подключение к серверам Google.
2. Реализована функция WaitForInternet(), которая в цикле ждет положительного результата CheckInternetConnection(). При его получении функция завершается и программа продолжает выполнение.
3. Обработка ошибок включает в себя случай отсутствия интернета.

logger.go: 
1. Реализована функция InitLogger:
    - читает путь к запущенному exe файлу программы
    - формирует путь к log файлу, добавляя его имя в конец пути
    - открывает log-файл или создает его в полученной ранее директории
    - указывает пакету log писать в созданный log-файл с указанием даты и времени
2. Реализована функция LogFatal(), при вызове которой в log-файл пишется сообщение о фатальной ошибке и программа завершается кодом 1.
3. Реализована функция LogError(), при вызове которой в log-файл пишется сообщение об ошибке, программа продолжает свое выполнение.
4. Реализована функция Close(), при вызове которой работа с log-файлом завершается(закрытие файла). Она завершается при завершении синхронизации.
5. Обработка ошибок включает в себя случаи ошибки получения путь к exe, ошибку создания(открытия) log-файла.

sync.go: 
1. Реализована функция LatestFile(), принимающая путь к директории в системе и возвращающая ошибку и полный путь до самого нового файла в директории(файла, в который были внесены самые последние изменения). Обработка ошибок включает в себя случаи ошибки чтения локальной директории, получения информации о файлах в ней.
2. Реализована функция Upload(), принимающая созданного в пакете client структуру клиента, путь к файлу для синхронизации и путь к заменяемому файлу на сервере. Выполняет удаление старого и установку нового файла на сервер. Возвращает ошибку. Обработка ошибок включает в себя случаи ошибки удаления файла на сервере, открытия локальной директории, записи файла на сервер.

client.go:
1. Реализована функция NewClient(), принимающая url, логин и пароль, возвращает структуру клиента из библиотеки gowebdav и ошибку.
2. Функция Connect() принимает клиента, полученного из функции NewClient(), выполняет подключение к серверу и возвращает ошибку.
